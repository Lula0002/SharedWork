# Contributing to Brain

## Adding content

1. Branch off `main` with a descriptive name: `add/onboarding-checklist`, `update/code-review-norms`
2. Place your file in the right folder:
   - **New collaboration norm?** → `collaboration/`
   - **New coding standard?** → `standards/`
   - **New step-by-step workflow?** → `processes/`
3. Write in plain markdown. One idea per file.
4. Open a PR and get at least one review before merging.

## Editing existing content

Prefer small, focused PRs — a single clarification or correction is easier to review than a sweeping rewrite.

## Removing content

Don't delete without discussion. Open a PR with an explanation of why the content no longer applies.
