# Incidents

## During an incident

1. Declare it — post in the team channel so everyone knows.
2. Assign a single owner to coordinate the response.
3. Fix first, understand second. Revert if unsure.

## After an incident

- Write a brief post-mortem within 48 hours.
- Focus on systems and processes, not individuals.
- Add any action items as tasks with a named owner and deadline.

## Post-mortem format

```
## What happened
## Timeline
## Root cause
## What we're doing about it
```
