# Decision Log

Significant decisions that affect how the project is built or run.
Add an entry when a call is made that would be confusing to future team members without context.

For larger architectural decisions, use the RFC process (`rfc-template.md`) and link to the PR here.

---

## Template

| Field | Value |
|---|---|
| **Date** | YYYY-MM-DD |
| **Decision** | One sentence — what was decided |
| **Why** | The reason — constraint, data, or tradeoff that drove it |
| **Alternatives rejected** | What else was considered and why it lost |
| **Owner** | Who is accountable for following through |
| **RFC / PR** | Link if applicable |
